# Workflow Orchestrator

A local AI Developer Workflow Orchestrator that automates repetitive actions between development tools.

## Overview

The Workflow Orchestrator is a Python CLI application that helps developers execute AI-assisted coding tasks by automating interactions between:

- **ChatGPT** — Prompts generated manually by the user
- **Freebuff** — AI code generation tool
- **VS Code** — Code editor
- **Brave Browser** — Web browser for deployment pages
- **Terminal** — Command execution
- **Git & GitHub** — Version control and remote hosting
- **Render** — Cloud hosting platform
- **Vercel** — Frontend deployment platform

## Features

- ✅ Open Brave Browser with specific URLs (GitHub, Render, Vercel)
- ✅ Open VS Code with project directories or specific files
- ✅ Copy prompts to system clipboard
- ✅ Execute terminal commands with output capture
- ✅ Git automation (status, add, commit, push)
- ✅ Deployment page automation (Render, Vercel)
- ✅ Centralized logging to file and console
- ✅ External configuration (no hardcoded values)
- ✅ Graceful error handling
- ✅ Interactive CLI menu

## Installation

### Prerequisites

- Python 3.12 or higher
- pip (Python package manager)

### Steps

1. **Clone the repository**

   ```bash
   git clone https://github.com/yourusername/workflow_orchestrator.git
   cd workflow_orchestrator
   ```

2. **Create a virtual environment** (recommended)

   ```bash
   python -m venv venv
   
   # Windows
   venv\Scripts\activate
   
   # macOS / Linux
   source venv/bin/activate
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

4. **Configure the application**

   ```bash
   python main.py
   ```

   From the main menu, select option **11** to open the configuration menu.

   Alternatively, edit `data/config.json` directly:

   ```json
   {
     "brave_executable_path": "C:/Program Files/BraveSoftware/Brave-Browser/Application/brave.exe",
     "vscode_executable_path": "code",
     "default_project_directory": "C:/Users/you/projects/my-app",
     "github_repository_url": "https://github.com/you/my-app",
     "render_dashboard_url": "https://dashboard.render.com/web/svc-xxxxx",
     "vercel_dashboard_url": "https://vercel.com/you/my-app",
     "freebuff_command": "freebuff"
   }
   ```

## Usage

Run the application:

```bash
python main.py
```

### Main Menu

```
=====================================
   AI Developer Workflow Orchestrator
=====================================

  1.  Copy Prompt to Clipboard
  2.  Open Freebuff
  3.  Open VS Code
  4.  Git Push
  5.  Open GitHub
  6.  Open Render Dashboard
  7.  Open Render Logs
  8.  Open Vercel Dashboard
  9.  Open Website / Deployment URL
 10.  Run Custom Terminal Command
 11.  Configuration
 12.  Run Full Workflow
 13.  View History
 14.  Exit
```

### Full Workflow

Select option **12** to run the complete automated workflow:

1. ✅ Enter your task description
2. ✅ Paste the prompt generated by ChatGPT
3. ✅ Prompt is automatically copied to clipboard
4. ✅ Freebuff is launched
5. ✅ Wait for your confirmation that Freebuff finished
6. ✅ VS Code opens with the project
7. ✅ Git status is checked and changes are committed/pushed
8. ✅ GitHub repository opens in browser
9. ✅ Render or Vercel dashboard opens
10. ✅ Deployment URL opens in browser
11. ✅ Success/failure is reported and logged

## Configuration

All configurable values are stored in `data/config.json`. Nothing is hardcoded in the source code.

| Key                     | Description                                      |
| ----------------------- | ------------------------------------------------ |
| `brave_executable_path` | Path to Brave browser executable                 |
| `vscode_executable_path`| Path to VS Code executable (default: `code`)     |
| `default_project_directory` | Default project directory for Git/VS Code    |
| `github_repository_url` | URL of the GitHub repository                     |
| `render_dashboard_url`  | URL of the Render service dashboard              |
| `vercel_dashboard_url`  | URL of the Vercel project dashboard              |
| `freebuff_command`      | Command to launch Freebuff                       |

## Logging

Logs are written to both:
- **Console**: INFO level and above
- **File**: DEBUG level and above, stored in `logs/workflow_YYYYMMDD_HHMMSS.log`

## Development

### Code Style

- Follows PEP 8 guidelines
- Uses type hints throughout
- Modular design with Single Responsibility Principle
- No global variables
- Dataclasses for structured data
- pathlib instead of os.path where possible

### Adding New Modules

1. Create a new file in the `modules/` directory
2. Import it in `main.py`
3. Add the corresponding menu option
